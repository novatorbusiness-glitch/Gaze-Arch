git add . 
- Добавляем все измененные файлы в индекс для сохранения

git commit -m "fix: restore styles and content" 
- Фиксируем изменения локально с комментарием.

git push
- Выгружаем твой локальный коммит на сервер GitHub.

git status
- Если не понимаешь, какие файлы изменены и что готово к отправке.

git reset --hard origin/main
- Если ИИ снова «накосячил» и ты хочешь мгновенно вернуть рабочую версию из облака. (Внимание: эта команда безвозвратно удалит все твои несохраненные локальные правки).

git log --oneline
- Чтобы увидеть список последних сохранений (коммитов).


git remote set-url origin https://github.com/novatorbusiness-glitch/Gaze-Arch.git
- Cинхронизации если поменял название на гит например.

git add CNAME
git commit -m "domain lock"
git push origin main
- GitHub видит его и автоматически прописывает этот адрес в настройках Custom Domain. Теперь, сколько бы ты ни обновлял код, GitHub будет знать: «Этот проект принадлежит домену gaze-arch.ru» и не будет его сбрасывать.
